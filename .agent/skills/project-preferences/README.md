# project-preferences

Repository-level preference skill

## What it enforces

- Full QA/testing plus security-focused validation for every code or feature change.
- Clean, reactive UI behavior for user-facing work.
- Iterative test/fix/retest loops until solution readiness is validated.

## Usage

This skill is auto-defaulted by `AGENTS.md` at the repository root.

## Customization for contributors

Contributors can extend agent behavior for their workflow by editing:
- `.agent/skills/project-preferences/SKILL.md`

Recommended use:
- Add concrete instructions for coding style, review expectations, testing depth, or delivery format.
- Keep instructions specific and actionable so coding agents can follow them consistently across requests.

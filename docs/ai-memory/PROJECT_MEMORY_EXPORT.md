## Setup, Run, And Test Commands
- Initial env setup:
  - Windows PowerShell: `Copy-Item .env.example .env`
  - macOS/Linux: `cp .env.example .env`
- First-time setup/start:
  - Windows PowerShell: `.\start.ps1`
  - Windows cmd: `start.cmd`
  - macOS/Linux: `chmod +x start.sh verify.sh reset.sh && ./start.sh`
- Verify:
  - Windows PowerShell: `.\verify.ps1`
  - Windows cmd: `verify.cmd`
  - macOS/Linux: `./verify.sh`
- Unit tests only:
  - Windows: `.\test.cmd`
  - macOS/Linux: `./test.sh`
- One-command setup+verify+probe:
  - Windows: `.\doctor.cmd`
  - macOS/Linux: `chmod +x doctor.sh && ./doctor.sh`
- Daily no-rebuild gateway start (from `vendor/openclaw`):
  - `docker compose -p openclaw-easy --env-file .env -f docker-compose.safe.yml up -d openclaw-gateway`
- Reset:
  - Basic: `.\reset.ps1` or `./reset.sh`
  - Full cleanup: `.\reset.ps1 -Full` or `./reset.sh --full`
